Expanda o sistema atual adicionando uma quarta página completa, dedicada ao gerenciamento de Produtos, aplicando os conceitos de navegação por abas e mantendo a arquitetura profissional MVC que acabaram de estruturar. O primeiro passo será abrir o arquivo index.html para modificar a interface visual; ali, eles devem criar um novo botão no menu lateral escrito "Produtos", seguindo o mesmo padrão estético dos anteriores, e logo em seguida estruturar uma nova seção de conteúdo (<section>) com o identificador único id="page-produtos", lembrando-se de incluir a classe hidden do Tailwind CSS para que essa tela comece oculta. Com a interface desenhada, os alunos devem passar para a lógica de dados abrindo o arquivo Model.js, onde criarão uma nova lista contendo três ou quatro produtos fictícios, incluindo atributos simples como nome do item, preço e quantidade em estoque. Na sequência, o trabalho vai para o arquivo View.js, onde eles deverão desenvolver a função responsável por capturar esses dados do modelo e renderizá-los dinamicamente dentro da nova seção do HTML, utilizando componentes modernos do Tailwind como tabelas limpas ou cartões com o efeito visual de Glassmorphism, além de garantir que o evento de clique do novo botão do menu seja ouvido para alternar a visibilidade das páginas e destacar o botão ativo. Por fim, para amarrar todo o fluxo, os alunos abrirão o arquivo Controller.js para atualizar o intermediário do sistema, programando o "garçom" da aplicação para buscar a lista de itens no modelo e ordenar que a visualização os desenhe na tela assim que o usuário clicar na aba correspondente.
